ADD RefRag pipeline (~30x RAG latency reduction)
Compress-Sense-Expand · ~30x faster · same answer quality
▶
Current CognitiveRAG
7-stage pipeline reruns full retrieval (2 rounds × 3s each) every query. Self-verification always fails (score=0.0) so round 2 is always triggered. ~34 seconds total for displacement query.
With RefRag

The key architectural insight
AetherForge currently calls Ollama over HTTP (Python → localhost:11434).
ruvllm eliminates that network hop by running inference in Rust,
called directly from Tauri. Latency: HTTP overhead (~5ms) → zero.

Similarly, ChromaDB is called from Python over a local socket.
RuVector-core in Tauri calls it from Rust at <100µs.
The Python layer only handles orchestration logic, not hot-path inference.
